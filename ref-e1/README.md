# REF-E1 — Refactoring-Katalog

Einsendeaufgabe REF-E1 (Softwaretechnik, BHT). Grundlage: der Refactoring-Katalog
von Martin Fowler (refactoring.com/catalog).

## Aufgabe

1. Eindrücke zum Katalog — beeindruckende, überflüssige, unverstandene Refactorings.
2. Genutzte IDE und ihre Refactoring-Unterstützung.
3. Eigenes, nicht triviales Code-Beispiel mit Vorher/Nachher und Fowler-Refactoring.
4. Nicht triviales Refactoring mit generativer KI samt Erfahrungsbericht.

## Inhalt

- `katalog-eindruecke.md` — Teil 1: Eindrücke zum Fowler-Katalog.
- `ide-refactorings.md` — Teil 2: VS Code und unterstützte Refactorings.
- `beispiel-eigenes.md` — Teil 3: eigenes Beispiel (Replace Conditional with Polymorphism).
- `beispiel-genai.md` — Teil 4: Refactoring mit KI (Extract Function + Decompose Conditional).

## Kontext

Sprache JavaScript, IDE VS Code. Die Code-Beispiele stammen aus der Domäne des
IT-Service-Monitors (durchgehendes SWT-Projekt).
