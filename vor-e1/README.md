# VOR-E1 — Vorgehensmodell für Projekt wählen

Einsendeaufgabe VOR-E1 (Softwaretechnik, BHT). Betrachtetes Projekt:
**IT-Service-Monitor** — das durchgehende SWT-Projekt, ein Werkzeug zur
Überwachung von Diensten per HTTP-, TCP- und DNS-Checks.

## Aufgabe

1. Vorgehensmodell wählen (Scrum oder Kanban).
2. Über Scrum / Kanban und Tools recherchieren.
3. Beweisbar mit einem Tool experimentieren.
4. Die Spotify-Videos ansehen und Beobachtungen festhalten.
5. Ersten Backlog erstellen und die erste Iteration planen.

## Entscheidungen

- **Vorgehensmodell:** Scrum (Begründung in `vorgehensmodell.md`).
- **Tool:** GitHub Issues + Milestones im Repo RDBHT/SWT — kein separates
  Projekt, Issues öffentlich verlinkbar (beweisbar).

## Inhalt

- `vorgehensmodell.md` — Teil 1: Modellwahl, Begründung, Scrum-Setup.
- `recherche.md` — Teil 2: Scrum, Kanban, Vergleich, Tool-Recherche.
- `tool-experiment.md` — Teil 3: Experiment mit GitHub Issues.
- `spotify-beobachtungen.md` — Teil 4: Auswertung der Spotify-Videos.
- `backlog.md` — Teil 5: Product Backlog und Sprint-1-Planung.
