Vibe & AI Coding

AI-gestützte Softwareentwicklung in drei Stufen

Durchgängiges Projekt: ein IT-Service-Monitor — ein Tool, das Services per Health-Check überwacht (HTTP, TCP-Port, DNS), den Verlauf speichert und bei Ausfällen alarmiert. Dasselbe Konzept in drei Stufen: vom schnellen Vibe-Coding bis zur modular gebauten, vollständig verstandenen App.

Teil A · Teil B · Teil C

Teil A — UI vibe-coden

UI-Entwurf für den Service-Monitor mit Google Stitch. Überlegte Prompts in mehreren Iterationen, dokumentiert im Repo.

in Bearbeitung

Teil B — Pet-Project vibe-coden

Lauffähige Vibe-Coding-Version des Monitors mit Lovable — funktional, aber bewusst noch nicht modular geschnitten.

in Bearbeitung

Teil C — Modulare, verteilte App

Dasselbe Konzept "richtig" gebaut: sechs separate Module (checker, scheduler, store, alerting, dashboard-ui, mock-service), entwickelt mit Claude Code (CLI), step-by-step über Spezifikations-MD-Files. Drei Check-Typen via Strategy Pattern: HTTP, TCP-Port, DNS.
