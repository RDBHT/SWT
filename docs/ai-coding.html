Vibe & AI Coding

AI-gestützte Softwareentwicklung in drei Stufen

Durchgängiges Projekt: ein IT-Service-Monitor — ein Tool, das Services per Health-Check überwacht (HTTP, TCP-Port, DNS), den Verlauf speichert und bei Ausfällen alarmiert. Dasselbe Konzept in drei Stufen: vom schnellen Vibe-Coding bis zur modular gebauten, vollständig verstandenen App.

Teil A — UI vibe-coden

Ergebnisse je Iteration

Fünf aufeinander aufbauende Prompts in Claude Design — jede Version als eigenständige HTML-Datei exportiert und hier live eingebettet. Prompt-Wortlaute je Iteration in prompts.md.

Iteration 1 — Design-Sprache

Layout, Typografie, Status-Farbskala (OK / degraded / down / unbekannt) und Basis-Bausteine (Status-Badge, Service-Karte, Tabelle) — noch keine einzelnen Screens.

Version in neuem Tab öffnen ›

Iteration 2 — Dashboard

Hauptscreen mit Service-Liste, Statusübersicht (OK / degraded / down) und Filter nach Status.

Version in neuem Tab öffnen ›

Iteration 3 — Service-Detail

Detailansicht eines Service mit Konfiguration (URL/Host/Port, Check-Typ, Intervall), hervorgehobenem Status und Verlauf der letzten Checks.

Version in neuem Tab öffnen ›

Iteration 4 — Service-Formular

Anlegen/Bearbeiten eines Service mit typabhängigen Feldern (HTTP / TCP-Port / DNS), Prüfintervall und sichtbarer Validierung.

Version in neuem Tab öffnen ›

Iteration 5 — Empty State + Politur

Leerzustand des Dashboards und Konsistenz-Runde über alle Screens. (offen)
