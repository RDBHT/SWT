Mission

Diese Seite dokumentiert die Einsendeaufgaben des Moduls Softwaretechnik für die Modulbetreuung. Sie bietet einen zentralen, navigierbaren Zugang zu allen Abgaben und ergänzt das reine Repo um interaktive Artefakte (z. B. ein lauffähiges Requirements-Tool statt eines Screenshots).

Abgaben

  • DVC-E1 — Versionsverwaltung mit Git abgegeben
  • REQ-E1 — Projektaufgabe Requirements abgegeben
  • ANA-E1 — Checkliste und Analyse abgegeben
  • DDD — Domain-Driven Design (Präsenzaufgabe) abgegeben
  • AI-Coding — Vibe & AI Coding (Präsenzaufgabe) abgegeben
  • UML-E1 — Eigenes UML-Werkzeug, fünf Diagramme abgegeben
  • VOR-E1 — Vorgehensmodell wählen (Scrum) abgegeben
  • REF-E1 — Refactoring-Katalog abgegeben
  • BUI-E1 — Buildmanagement (Maven + GitHub Actions) abgegeben
  • TST-E1 — Projektaufgabe Testen (Unit-Tests, TDD, Mocking) abgegeben
  • MET-E1 — Metriken (PMD + Checkstyle, McCabe von Hand) abgegeben
  • ARC-E1 — Finale Architektur (Komponentendiagramm, IT-Service-Monitor) abgegeben
  • ARC-E2 — Architecture Communication Canvas (Alerting-Pipeline) abgegeben
